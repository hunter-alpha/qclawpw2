# OpenClaw 内容生产系统 SOP

## 1. 选题机制

### 1.1 选题来源优先级
1. **官方渠道** (权重: 10)
   - OpenClaw 官方文档更新
   - GitHub 仓库新功能发布
   - 官方博客技术文章

2. **社区热点** (权重: 8)
   - Reddit r/OpenClaw 热门讨论
   - Discord/Slack 社区高频问题
   - Stack Overflow 相关问题

3. **搜索趋势** (权重: 7)
   - Google Trends 上升关键词
   - Ahrefs/SEMrush 搜索量增长
   - 竞争对手内容缺口

4. **用户反馈** (权重: 6)
   - 网站搜索查询分析
   - 社区提问统计
   - 邮件咨询主题

### 1.2 内容类型判断矩阵

| 搜索意图 | 关键词特征 | 内容类型 | 页面模板 |
|---------|-----------|----------|----------|
| 导航型 | "OpenClaw login", "OpenClaw docs" | 工具/资源页 | /tools/[slug] |
| 信息型 | "what is OpenClaw", "how OpenClaw works", "什么是 QClaw", "QClaw 是什么" | 概念解释页 | /openclaw/ 或 /qclaw/ |
| 教程型 | "OpenClaw tutorial", "QClaw 怎么用", "QClaw 使用教程" | 分步教程 | /tutorials/[slug] |
| 问题型 | "OpenClaw error", "QClaw 常见问题" | 故障排除/FAQ | /questions/[slug] |
| 对比型 | "QClaw vs OpenClaw", "QClaw 和 OpenClaw 有什么区别" | 对比分析 | /compare/[slug] |
| 商业型 | "QClaw 使用场景", "QClaw 适合谁" | 解决方案 | /use-cases/[slug] |

### 1.3 核心关键词集群 (Topical Clusters)

**1. QClaw 意图词 / 问题词集群 (高优先级流量入口)**
- 核心词：什么是 QClaw, QClaw 是什么, 腾讯 QClaw 是什么, QClaw 是做什么的
- 问题词：QClaw 能做什么, QClaw 怎么用, QClaw 如何上手, QClaw 常见问题
- 场景词：QClaw 适合谁, QClaw 使用场景
- 对比词：QClaw 和 OpenClaw 什么关系, QClaw 和 OpenClaw 有什么区别, QClaw vs OpenClaw

**2. OpenClaw 核心认知集群 (长期权威积累)**
- 核心词：OpenClaw framework, what is OpenClaw
- 教程词：OpenClaw tutorial, build OpenClaw skill

### 1.3 选题评分标准

**总分 = 搜索量 × 0.4 + 商业价值 × 0.3 + 技术价值 × 0.2 + 竞争难度 × 0.1**

- **搜索量** (1-10分): 基于关键词月搜索量
- **商业价值** (1-10分): 转化潜力评估
- **技术价值** (1-10分): 对社区的技术贡献
- **竞争难度** (1-10分): 现有内容质量评估

## 2. 资料搜集机制

### 2.1 信息源优先级

1. **一级来源** (必须引用)
   - OpenClaw 官方文档
   - 源代码注释
   - 官方示例项目

2. **二级来源** (优先引用)
   - 核心开发者博客

## 3. 内容生成规则

### 3.1 内容结构标准

**必须包含的模块**:
1. **Answer-first 摘要** (40-80字)
2. **定义模块** (What is it?)
3. **适用场景** (When to use?)
4. **核心优势** (Why choose?)
5. **实现步骤** (How to implement?)
6. **最佳实践** (Best practices)
7. **常见问题** (FAQ)
8. **相关资源** (Further reading)

### 3.2 原创性要求

**禁止行为**:
- ❌ 直接翻译英文文档
- ❌ 复制粘贴代码示例
- ❌ 改写现有教程句式
- ❌ 使用通用模板套话

**鼓励行为**:
- ✅ 添加个人实践经验
- ✅ 提供独特视角分析
- ✅ 创建原创代码示例
- ✅ 设计可视化图表

### 3.3 内容质量检查清单

**技术准确性**:
- [ ] 所有代码示例经过测试
- [ ] 版本号与当前发布一致
- [ ] 配置参数真实可用
- [ ] 错误处理逻辑完整

**内容深度**:
- [ ] 至少包含3个实际案例
- [ ] 提供性能对比数据
- [ ] 包含常见错误及解决方案
- [ ] 给出扩展应用建议

## 4. 多语言本地化规则

### 4.1 翻译策略矩阵

| 内容类型 | 翻译方式 | 本地化程度 | 示例 |
|---------|----------|------------|------|
| 技术术语 | 保持英文 | 0% | "API", "SDK", "JSON" |
| 概念解释 | 意译为主 | 70% | "skill" → "技能" (中文) / "Fähigkeit" (德文) |
| 使用场景 | 文化适配 | 90% | 用本地案例替换 |
| 营销文案 | 完全重写 | 100% | 根据本地市场调整 |

### 4.2 本地化检查要点

**语言层面**:
- [ ] 技术术语一致性
- [ ] 语法结构符合本地习惯
- [ ] 度量单位本地化
- [ ] 日期格式适配

**文化层面**:
- [ ] 案例场景本土化
- [ ] 商业语境适配
- [ ] 法律合规检查
- [ ] 敏感内容审查

## 5. 人工审核流程

### 5.1 三级审核体系

1. **技术审核** (开发者)
   - 代码准确性验证
   - 技术概念检查
   - 最佳实践符合度

2. **语言审核** (本地化专家)
   - 翻译质量评估
   - 文化适应性检查
   - 术语一致性验证

3. **SEO审核** (SEO专家)
   - 关键词优化检查
   - 结构化数据验证
   - 内链策略执行

### 5.2 审核评分表

| 维度 | 权重 | 评分标准 | 合格线 |
|------|------|----------|--------|
| 技术准确性 | 30% | 1-10分 | ≥8分 |
| 内容原创性 | 25% | 1-10分 | ≥7分 |
| 用户体验 | 20% | 1-10分 | ≥8分 |
| SEO优化 | 15% | 1-10分 | ≥7分 |
| 本地化质量 | 10% | 1-10分 | ≥8分 |

**总分 ≥ 7.5分 方可发布**

## 7. 内容更新机制

### 7.1 定期更新周期
- **技术文档**: 每月检查更新
- **教程内容**: 每季度全面审查
- **案例研究**: 每半年更新案例
- **性能数据**: 每季度重新测试

### 7.2 更新触发条件
- 软件版本升级
- 用户反馈错误
- 搜索排名下降
- 竞争对手内容更新

### 7.3 版本控制规范
```
内容版本: [主版本].[次版本].[修订]
- 主版本: 重大技术更新
- 次版本: 新增功能或案例
- 修订: 错误修正或小优化
```

## 8. 性能监控指标

### 8.1 内容表现KPI
- **搜索排名**: 目标关键词前3页占比
- **点击率**: SERP点击率 > 5%
- **停留时间**: 平均页面停留 > 2分钟
- **跳出率**: < 40%
- **社交分享**: 每月分享次数增长

### 8.2 用户参与度
- **评论数量**: 每篇文章 ≥ 3条评论
- **问题反馈**: 响应时间 < 24小时
- **社区讨论**: 相关话题提及次数

### 8.3 转化指标
- **邮件订阅**: 内容页转化率 > 2%
- **技能试用**: 教程页转化率 > 5%
- **社区加入**: 整体转化率 > 1%
